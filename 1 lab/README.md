# Notes API — Лабораторная работа 1: Docker

Простое REST API для управления заметками на Go + PostgreSQL.

## Эндпоинты

| Метод | Путь       | Описание              |
|-------|------------|-----------------------|
| GET   | /health    | Проверка работоспособности |
| GET   | /notes     | Получить все заметки  |
| POST  | /notes     | Создать заметку       |

**Пример создания заметки:**
```bash
curl -X POST http://localhost:8080/notes \
  -H "Content-Type: application/json" \
  -d '{"title":"Привет","body":"Это моя первая заметка"}'
```

## Запуск через docker-compose

```bash
docker-compose up --build
```

## Ответы на вопросы к защите

### 1. Что такое контейнеризация? Как она упрощает развертывание и управление приложениями?

**Контейнеризация** — это технология изоляции приложения вместе со всеми его зависимостями (библиотеки, конфиги, среда выполнения) в единый переносимый пакет — **контейнер**.

Контейнер использует ядро операционной системы хоста, но при этом изолирован от других процессов с помощью механизмов Linux (namespaces, cgroups).

**Упрощение развертывания:**
- Принцип «написал один раз — работает везде»: контейнер одинаково ведёт себя на ноутбуке разработчика, на CI-сервере и в продакшне.
- Нет конфликтов зависимостей между приложениями на одном хосте.
- Быстрый запуск (секунды против минут у виртуальных машин).
- Лёгкое масштабирование: запустить 10 копий контейнера так же просто, как одну.

---

### 2. Что такое Docker и для чего он используется?

**Docker** — открытая платформа для разработки, доставки и запуска приложений в контейнерах.

Docker предоставляет:
- **Docker Engine** — демон для управления контейнерами.
- **Docker CLI** — интерфейс командной строки (`docker build`, `docker run`, и т.д.).
- **Docker Hub** — публичный реестр образов.
- **docker-compose** — инструмент для запуска многоконтейнерных приложений.

Используется для: локальной разработки, CI/CD пайплайнов, деплоя микросервисов, изоляции окружений.

---

### 3. Что такое образ в Docker? Как собрать образ?

**Образ (image)** — это неизменяемый (read-only) шаблон, содержащий файловую систему с приложением и его зависимостями. Образ состоит из слоёв (layers), каждый из которых соответствует инструкции в `Dockerfile`.

**Сборка образа:**
```bash
docker build -t my-app:latest .
```
Docker читает `Dockerfile`, последовательно выполняет инструкции (`FROM`, `COPY`, `RUN`, и т.д.) и создаёт образ.

---

### 4. Что такое контейнер в Docker? Чем контейнер отличается от образа?

**Контейнер** — это запущенный (работающий) экземпляр образа. Над слоями образа Docker добавляет тонкий writable-слой, куда записываются все изменения в процессе работы контейнера.

| | Образ | Контейнер |
|---|---|---|
| Состояние | Статичный (read-only) | Живой процесс (read-write слой сверху) |
| Аналогия | Класс в ООП | Объект (экземпляр класса) |
| Хранение | В реестре / локально | В памяти / на диске хоста |

Из одного образа можно запустить любое количество контейнеров.

---

### 5. Что такое реестр контейнеров в Docker? Зачем он нужен?

**Реестр (registry)** — это хранилище Docker-образов, откуда образы можно загружать (`pull`) и куда отправлять (`push`).

**Зачем нужен:**
- Совместное использование образов внутри команды.
- Деплой: сервер забирает образ из реестра и запускает контейнер.
- Версионирование образов через теги (`v1.0`, `latest`).

Публичный реестр — **Docker Hub** (`hub.docker.com`). Существуют также приватные реестры: GitHub Container Registry, GitLab Registry, AWS ECR, и т.д.

---

### 6. Как отправить образ в реестр контейнеров?

```bash
# 1. Авторизоваться
docker login

# 2. Собрать образ с именем вида <username>/<repo>:<tag>
docker build -t 1lostsun/docker-lab:latest .

# 3. Отправить в реестр
docker push 1lostsun/docker-lab:latest
```

После этого образ доступен публично по адресу `docker.io/1lostsun/docker-lab`.

---

### 7. Что такое docker-compose и в чём его преимущества?

**docker-compose** — инструмент для описания и запуска многоконтейнерных приложений с помощью одного YAML-файла (`docker-compose.yml`).

**Преимущества:**
- Всё описано декларативно в одном файле — удобно хранить в репозитории.
- Один командой (`docker-compose up`) поднимается всё приложение со всеми сервисами.
- Автоматически создаётся общая сеть для сервисов.
- Поддержка зависимостей между сервисами (`depends_on`), переменных окружения, volumes.

---

### 8. Как docker-compose управляет сетевым взаимодействием между контейнерами?

По умолчанию docker-compose создаёт для проекта отдельную **bridge-сеть**. Все сервисы, описанные в `docker-compose.yml`, автоматически подключаются к этой сети.

Внутри этой сети каждый сервис доступен по своему **имени** (как DNS-имя). Например, сервис `app` обращается к сервису `db` просто по хосту `db` на порту `5432` — без явных IP-адресов.

```yaml
environment:
  PG_HOST: db   # имя сервиса из docker-compose.yml
```

Можно также задавать явные сети через секцию `networks:`.

---

### 9. Что такое проброс портов в Docker?

**Проброс портов (port mapping/publishing)** — это механизм, при котором порт внутри контейнера становится доступен на хост-машине.

Синтаксис: `HOST_PORT:CONTAINER_PORT`

```yaml
ports:
  - "8080:8080"   # запросы на localhost:8080 → в контейнер на :8080
  - "5432:5432"   # запросы на localhost:5432 → в контейнер на :5432
```

Без проброса контейнер изолирован и не доступен извне (только внутри Docker-сети).
